## 概览
**MyECS**是我用C++写的简易ECS引擎，采用SparseSet实现。接口与**entt**相似。

基本是**entt**的基础功能的复刻。

需要**C++20**和**MSVC**。

[**MYECS**](https://github.com/Flaaax/NoitaSurvivor) 参看 **src/ecs**

[**entt**](https://github.com/skypjack/entt)

## 入口
### MyECS的抽象接口。

```cpp
myecs::Registry reg;
```
声明一个**Registry**。它管理你的**实体句柄**，并储存**组件**。

它也是整个**ECS**系统的入口。
```cpp
auto e = reg.create();
```
创建一个实体。你拿到的是**实体句柄**。

它**轻量，可复制，持久有效**。不携带任何数据。
```cpp
auto& pos = reg.emplace<Position>(e, 1.f, 2.f);
```
为实体**e**创建一个**Position**组件。返回组件的引用。

**Position**是你的自定义类型。它表示：此实体具有的某种属性。
```cpp
struct Position {
    float x{};
    float y{};
};
```
**Position**不关心它如何被使用，它的持有者是谁。它只携带一些数据。

```cpp
auto& pos = reg.get<Position>(e);
```
访问实体**e**绑定的**Position**组件。

这进一步说明：**e**只是一个句柄，真正的持有关系在**Registry**中。
```cpp
auto& v = reg.emplace<Velocity>(e);   // 添加一种新的组件。
reg.destroy<Position>(e);   // 从 e 移除此组件。
reg.destroy(e);             // 移除 e 以及它附带的所有组件。
// 此时 v 已失效
```
从**Resistry**移除实体后，这个句柄会自动失效。你可以这样判断：
```cpp
bool isValid = reg.valid(e);
bool hasVelocity = reg.has<Velocity>(e);
```
**hasVelocity**为**False**，因为已经不存在此实体。

你可以遍历所有持有某种组件的实体。
```cpp
const auto view = reg.view<Position, Sprite>();
for(auto [e, position, sprite] : view) {   
    // position 和 sprite 都是引用
    doSomeRender(sprite, position);

    if(auto options = reg.try_get<DebugRenderOptions>(e)){
        // options 是指针
        doSomeDebugRender(*options, position);
    }
}
```
在这里，你可以用**try_get**替代**has**来同时进行判断和获取。

### 与**OOP**的比较
继承式写法看起来更像这样
```cpp
for(const auto* entitiy : allEntities) {   
    entity->doSomeRender();
}
```
区别是，在**ECS**，你不需要为实体的新功能定制一个虚函数，不需要修改整个**Entity**基类，把所有数据都塞到里面。

你只需要在随便一个地方声明一种新的组件，然后说：“**我给这个实体添加此组件**” 

就搞定了。
